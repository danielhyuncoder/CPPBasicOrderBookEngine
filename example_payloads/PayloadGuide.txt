ORDERBOOK FUNCTION TYPES & JSON PAYLOAD FORMAT
=============================================

This system processes an array of JSON objects, where each object represents an instruction sent to the orderbook. Each instruction uses a FUNCTION field to determine what action to perform.

SUPPORTED FUNCTION TYPES:
-------------------------
- MARKET
- LIMIT
- MODIFY (with subtypes: KILL, QUANTITY, PRICE)
- DISPLAY (default fallback if FUNCTION is unknown)

Each JSON object should be structured as one of the following:

---------------------------------------------------
1. MARKET ORDER
---------------------------------------------------
Used to execute a market order at the best available price.

Required Fields:
- FUNCTION: "MARKET"
- DIRECTION: "BUY" or "SELL"
- QTY: Integer
- LOG: Optional, "OUT" for logging, otherwise "NONE" or omit



---------------------------------------------------
2. LIMIT ORDER
---------------------------------------------------
Used to place a limit order into the book.

Required Fields:
- FUNCTION: "LIMIT"
- DIRECTION: "BUY" or "SELL"
- QTY: Integer
- UID: Integer (Unique ID)
- PRICE: Float
- TIMESTAMP: Integer (Unix time or simulation time)
- LOG: Optional, "OUT" or "NONE"



---------------------------------------------------
3. MODIFY ORDER
---------------------------------------------------
Used to modify or cancel (kill) an existing order.

A. KILL ORDER
-------------
Required Fields:
- FUNCTION: "MODIFY"
- MODIFICATION: "KILL"
- ID: Integer (UID of the order)
- LOG: "OUT" or "NONE"



B. MODIFY QUANTITY or PRICE
---------------------------
Required Fields:
- FUNCTION: "MODIFY"
- MODIFICATION: "QUANTITY" or "PRICE"
- ID: Integer (UID of the order)
- VALUE: New quantity or price (integer or float)
- LOG: "OUT" or "NONE"




---------------------------------------------------
4. DISPLAY ORDER BOOK
---------------------------------------------------
Used to print/display the order book to the console.

Required Fields:
- RANGE: Integer (number of levels to show)

This is the fallback if FUNCTION is unrecognized.


---------------------------------------------------
NOTES
---------------------------------------------------
- All fields are case-sensitive.
- Unrecognized FUNCTION values default to DISPLAY.
- UID and ID should be integers.
- LOG field is optional and enables verbose output if set to "OUT".
- TIMESTAMP is assumed to be in long integer format.
